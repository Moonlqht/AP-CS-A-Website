Recursion

Programming is very dynamic - it allows the programmer to do virtually anything. Methods are sections of code that accomplish a specific task. They can also call other methods, including themselves. This concept of methods calling themself is called recursion.

Theory

There are three basic principles of recursion:

  1. A recursive method must call itself
  2. A recursive method must have a base case
  3. A recursive method must move the base case

In essence, what recursion does is that it splits a complex problem into smaller subproblems to solve. These subproblems are then solved(usually with recursion), splitting them down until we have something that's called a base case - a situation that will always be reached, and has a trivial solution. This means that each time the method is called, a smaller subproblem is being solved.

Recursion vs Iteration

Iterative implementations of a problem generally consist of a for loop. This loop then works its way up from the bottom, computing each step along the way. Recursion, on the other hand, solves the problem in terms of previous terms first, then goes ahead to compute the previous terms, which is a top-down approach compared to the bottom-up approach of iteration.

Below is an example of iterative Fibonacci code, compared to recursive Fibonacci code. 

// Recursive implementation
public int Fibonacci(int num){
    if(num == 0){
        return 0;
    }else if(num == 1){
        return 1;
    }
    return Fibonacci(num-1) + Fibonacci(num-2);
}
// Iterative implementation
public int Fibonacci(int num){
    int num1 = 0;
    int num2 = 1;
    for(int i = 0; i < num-1; i++){
        int temp = num2;
        num2 = num1 + num2;
        num1 = temp;
    }
    return num2;
}

The top-down vs bottom-up approach of the two is prevalent here, the iterative version started from 0 while the recursive one started from the term we want to calculate.

Common Mistakes

Common mistakes in code that uses recursion include:

  1. The absence of a base case
  2. Not moving towards the base case
  3. Assuming the compiler has unlimited memory

All of these issues generally have the same ending, when the compiler throws a StackOverflowException, which essentially means recursion depth exceeded, or that the compiler no longer has memory to allocate to running the program. Thinking from a high-level, it makes sense that this would happen, as if we don't reach the base case or take too long to reach it, too much memory will be used in each method call. The compiler will not allow the user to use infinite memory, causing it to crash the program.

Example

The following code segment is my code for one of the recursion exercises, where we were to use recursion to expand a word by adding hyphens between each pair of characters, using recursion. 

public String expand(String word){
    if(word.length() == 1){
        return word;
    }
    return word.charAt(0) + "-" + expand(word.substring(1));
}

In this code segment, we move towards the base case by making the word we're checking shorter. The base case is when the string is 1 character long, where we just return that string.
